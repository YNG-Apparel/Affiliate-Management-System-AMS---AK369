# 4. Non-Functional Requirements

## Overview
This document defines the quality attributes and operational expectations for the system.

## Requirements
- Reliability: The system should operate consistently with minimal downtime.
- Performance: Core operations should respond within acceptable time limits under normal load.
- Security: Authentication, authorization, and data protection must be enforced.
- Scalability: The platform should support future growth in users, affiliates, and transactions.
- Maintainability: Code and architecture should be easy to extend and maintain.
- Usability: Interfaces should be intuitive for both admins and affiliates.
