# 12. API Design

## Overview
This document outlines the expected API surface for the AMS.

## Suggested Endpoints
- POST /auth/register
- POST /auth/login
- GET /affiliates
- POST /affiliates
- GET /campaigns
- POST /campaigns
- GET /commissions
- POST /payouts

## Design Principles
- RESTful conventions
- Consistent response formats
- Proper authentication and role-based authorization
- Versioned APIs where appropriate
