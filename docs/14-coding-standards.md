# 14. Coding Standards

## General Guidelines
- Follow consistent naming conventions.
- Write clean, readable, and modular code.
- Add comments only where necessary.
- Keep functions focused on a single responsibility.
- Write tests for business-critical logic.

## Language-Specific Notes
- Use TypeScript where possible for frontend and backend clarity.
- Follow linting and formatting rules enforced by the repository.
- Keep dependencies updated and minimize unnecessary complexity.
